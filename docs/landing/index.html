How it works

Three rules your AI cannot break.

BotForge enforces these through the skill prompt, not style guides. The AI literally refuses to generate code that violates them.

01 — ARCHITECTURE

No monoliths.

Handlers, services, repositories, integrations — layered from day one. Business logic in a handler is a [blocker] finding.

02 — PRODUCTION

No drafts.

Docker, Postgres, Alembic, .env, webhook, deploy commands — in the very first generation. Zero "TODO: add later".

03 — EXTENSIBILITY

No breakage.

Diff-aware extension protocol. Adding the fifth feature is as easy as the first — and doesn't invalidate tests.
