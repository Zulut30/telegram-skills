# Usage Guide

## Prompt format

```
BotForge: [Lite|Pro|Media|SaaS]
Задача: <бизнес-описание>
Ограничения: <бюджет / хостинг / сроки / регуляторика>
Ответы на брифинг (опционально):
  1) ...
  2) ...
```

`BotForge: Pro` is the default. Media/SaaS add domain-specific modules.

## Four modes

| Mode | When to use | Differences from Pro |
|---|---|---|
| **Lite** | MVP за вечер, проверка гипотезы, одноразовый бот | SQLite + aiosqlite, polling, без Docker/Alembic, упрощённая структура |
| **Pro** (default) | Коммерческий бот для клиента / продукта | Полный стандарт |
| **Media** | Контент-боты, медиа-каналы, gated-доступ | + CMS-sync (WP/Notion), сегментированный broadcast, UTM-трекинг, реакции/опросы |
| **SaaS** | Подписки, VIP, billing, закрытые клубы | + plans/trials/proration, multi-provider payments, admin metrics, invite-link rotation |

## Session lifecycle

### 1. Create
```
BotForge: Pro
Задача: бот-витрина онлайн-курсов, оплата ЮKassa, выдача доступа в закрытый канал.
```
AI: 5 вопросов → ответы → ADR → tree → файлы → self-review → deploy.

### 2. Extend
```
Добавь реферальную программу: скидка 20% при приглашении друга, счётчик рефералов в профиле.
```
AI: план дельты (слои, файлы, миграции) → diff-код → self-review дельты.

### 3. Review
```
Review app/services/payment_service.py
```
AI: findings с тегами `[blocker] [major] [minor] [nit]`, ссылки `file:line`, предложенные правки.

### 4. Refactor monolith
```
Вот мой бот на 800 строк в main.py. Приведи в порядок.
[код]
```
AI: инвентаризация → план миграции → порядок PR-ов → риски и откат.

### 5. Deploy mode switch
```
Переведи на webhook, деплой на Fly.io.
```
AI: список изменений → конкретные команды → процедура отката.

## Anti-patterns (что не делать)

- Не просить «сгенерируй бота одним файлом» — skill откажется и предложит модульный вариант.
- Не вставлять токен бота в запрос — skill не будет хардкодить его в коде, но безопаснее не светить.
- Не смешивать несколько ботов в одном запросе — сделайте отдельную сессию.
- Не игнорировать ADR — это ваш шанс скорректировать архитектуру до генерации кода.

## Советы

- Отвечайте на брифинг **одним сообщением списком**: AI экономит токены и переходит к ADR быстрее.
- Перед крупными правками попросите `/review` — дешевле, чем переделывать.
- Для клиентских проектов сохраняйте ADR в `docs/ADR/` репозитория — это ваша защита на ретроспективах.
- Для повторяющихся ботов (агентство) заведите **корпоративный `system_prompt.txt`** с вашими конвенциями поверх BotForge.

## Интеграция с существующим кодом

Если проект уже есть:
```
Вот текущая структура: [tree]
Вот код handlers/start.py: [code]
BotForge: добавь FSM для сбора заявок без слома текущей архитектуры.
```
AI сохранит совместимость и предложит минимальную дельту.
