# Internationalization (i18n) — Reference

Мультиязычные боты. Подход BotForge — **gettext / .po файлы** (промышленный стандарт) или **fluent** (Mozilla), на выбор.

---

## Выбор подхода

| Подход | Когда |
|---|---|
| **JSON dict** | ≤2 языков, ≤50 строк, прототип |
| **gettext + Babel** | Production, ≤5 языков, переводчики-люди |
| **fluent** | Сложная плюрализация (ru/pl/cs), контекстные формы |

**Default BotForge:** gettext для production, JSON для Lite.

---

## Определение языка (приоритет)

1. Явно выбран пользователем (записано в `users.lang`)
2. `from_user.language_code` (ISO-639-1: `ru`, `en`, `uk`, ...)
3. Default из `settings.default_lang`

```python
# app/middlewares/i18n.py
from aiogram import BaseMiddleware
from app.utils.i18n import set_lang

SUPPORTED = {"ru", "en", "uk"}
DEFAULT = "en"

class I18nMiddleware(BaseMiddleware):
    async def __call__(self, handler, event, data):
        user = data.get("user")
        tg_user = getattr(event, "from_user", None)

        if user and user.lang:
            lang = user.lang
        elif tg_user and tg_user.language_code:
            lang = tg_user.language_code.split("-")[0]
        else:
            lang = DEFAULT

        if lang not in SUPPORTED:
            lang = DEFAULT

        set_lang(lang)         # contextvar
        data["lang"] = lang
        return await handler(event, data)
```

## gettext wrapper

```python
# app/utils/i18n.py
from contextvars import ContextVar
from gettext import translation
from pathlib import Path

_LOCALES = Path(__file__).parent.parent / "locales"
_current: ContextVar[str] = ContextVar("lang", default="en")

_cache: dict[str, "gettext.GNUTranslations"] = {}

def _load(lang: str):
    if lang not in _cache:
        _cache[lang] = translation("bot", localedir=str(_LOCALES),
                                   languages=[lang], fallback=True)
    return _cache[lang]

def set_lang(lang: str) -> None:
    _current.set(lang)

def _(msg: str) -> str:
    return _load(_current.get()).gettext(msg)

def ngettext(singular: str, plural: str, n: int) -> str:
    return _load(_current.get()).ngettext(singular, plural, n)
```

## Использование в handlers

```python
from app.utils.i18n import _

@router.message(Command("start"))
async def cmd_start(msg: Message) -> None:
    await msg.answer(_("welcome_message"))
```

## Extraction workflow

```bash
# pyproject.toml: [tool.babel] + babel как dev-dep
pybabel extract -o app/locales/messages.pot app/
pybabel init -i app/locales/messages.pot -d app/locales -D bot -l ru
pybabel init -i app/locales/messages.pot -d app/locales -D bot -l en
# редактируем .po в Crowdin / POEditor
pybabel compile -d app/locales -D bot
```

**CI-шаг:** `pybabel extract` + `pybabel update` → PR, если появились новые ключи.

## Правила для ключей

- Ключ = **английский текст сообщения** (дефолт) ИЛИ namespaced slug (`welcome.title`, `vip.expired`)
- BotForge рекомендует slug-подход: проще менять английский текст без ломки переводов.

## Плюрализация

```python
# ngettext автоматически применяет правильную форму для языка
msg = ngettext("{n} day left", "{n} days left", n).format(n=n)
```

Для русского нужны 3 формы: 1 день / 2-4 дня / 5-20 дней. gettext их корректно обрабатывает при правильных `.po` с Plural-Forms header.

## Форматирование дат/чисел

```python
from babel.dates import format_datetime
from babel.numbers import format_currency

format_datetime(dt, locale=lang)                 # "4 апр. 2026 г., 14:30"
format_currency(Decimal("299.00"), "RUB", locale=lang)  # "299,00 ₽"
```

## Anti-patterns

- ❌ `text = "Привет" if lang == "ru" else "Hello"` — не масштабируется
- ❌ F-string с переводом внутри: `f"{_('count')}: {n}"` — экранирование теряется
- ❌ Перевод в models / services — перевод в presentation layer (handlers / keyboards)
- ❌ Хранить переводы в БД — усложняет CI, мешает review переводчиков
- ❌ Автоперевод в рантайме (google translate API) — нестабильно и дорого
