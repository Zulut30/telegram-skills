Добавь в текущий Telegram-бот фичу: $ARGUMENTS

Следуй **Extension Protocol** BotForge:

1. **Inventory** — быстро прочитай текущее состояние: `app/handlers/`, `app/services/`, `app/models/` (используй Glob/Grep).
2. **План изменений** — выдай:
   - Слои, которые затрагиваем
   - Список новых файлов
   - Список изменяемых файлов
   - Нужна ли миграция БД
   - Сохраняется ли обратная совместимость (если нет — обоснуй)
3. **Миграция** — если модель данных меняется, сгенерируй Alembic-миграцию.
4. **Код** — только затронутые файлы, не трогай лишнее.
5. **Тесты** — добавь unit-тесты на новые сервисы.
6. **README** — обнови, если меняется поведение для оператора.
7. **Self-review дельты** — checkbox по затронутой функциональности.

**Правила:**
- Никаких breaking changes в публичных интерфейсах сервисов без обсуждения.
- Новая логика идёт в `services/`, данные — в `repositories/`, UI — в `handlers/` + `keyboards/`.
- Переиспользуй существующие middleware и фильтры.
- Если фича требует нового внешнего API — добавь клиента в `integrations/` с retry + timeout.
