Проведи рефакторинг монолита в архитектуру BotForge. Контекст: $ARGUMENTS

**Workflow:**

### 1. Inventory
Прочитай текущий код (Glob/Grep). Составь карту:
- Сколько файлов, строк
- Какие хэндлеры есть
- Где лежит бизнес-логика (в handlers? в main?)
- Где работа с БД (raw psycopg? raw SQL? чистый ORM?)
- Где секреты (захардкожены? в os.environ? в config.py?)
- Какие интеграции (requests? httpx? aiohttp?)
- Есть ли миграции / Docker / tests

### 2. План миграции (по шагам, без простоя)

Выдай таблицу:
| Шаг | Действие | Файлы | Риск | Откат |
|---|---|---|---|---|

Типичный порядок:
1. Извлечь конфиг → `app/config/settings.py` (pydantic-settings)
2. Ввести Alembic + baseline-миграция из текущей схемы
3. Завернуть модели в SQLAlchemy 2.x (Mapped[...]) — с обратной совместимостью
4. Вынести DB-доступ → `app/repositories/`
5. Вынести бизнес-логику → `app/services/`
6. Handlers становятся thin: parse → service → reply
7. Ввести middlewares (DI session, auth, throttling, errors)
8. Keyboards → фабрики в `app/keyboards/`
9. FSM → `app/states/`
10. Внешние API → `app/integrations/` с tenacity
11. Dockerfile + docker-compose
12. CI: ruff + mypy + pytest

### 3. Порядок PR-ов (атомарных)
Каждый шаг = отдельный PR, проходит тесты, мержится, дальше. Не большой PR на всё сразу.

### 4. Риски и откат
- Миграция данных → dry-run на реплике
- Изменение FSM → сессии пользователей могут сломаться (очистить Redis FSM или предусмотреть совместимость)
- Новые ENV-переменные → развёртка требует обновления `.env` на сервере

### 5. Если юзер говорит «rewrite»
Предложи **Strangler Fig pattern**:
- Новый проект рядом (`app_v2/`)
- Один хэндлер за раз переносится
- Фичфлаг в диспатчере: какой хэндлер обслуживает апдейт
- Когда все мигрированы — старый код удаляется

### 6. Не делай
- Не делай рефакторинг и новую фичу в одном PR
- Не ломай публичные команды бота (`/start`, `/help` — должны работать на каждом шаге)
- Не меняй поведение для пользователя на этапе рефакторинга (только архитектура)

### 7. Итог
После рефакторинга — полный self-review по BotForge checklist.
