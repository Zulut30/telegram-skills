TELEGRAM BOT API 9.6 — OFFICIAL CONSTRAINTS (enforced)
═══════════════════════════════════════════════════════════════════════
  Rate limits (core.telegram.org/bots/faq):
    • 1 msg/sec per user (private chat)
    • 20 msg/min per group
    • ~30 msg/sec broadcast (paid broadcasts up to 1000/sec)
    • BotForge throttles broadcasts to 25 msg/sec (safety margin)

  Error handling (must be distinct):
    • TelegramRetryAfter → asyncio.sleep(e.retry_after), retry once
    • TelegramForbiddenError → mark user blocked=true, NEVER retry
    • TelegramBadRequest → fix + no retry (validation)
    • TelegramUnauthorizedError → alert, rotate token
    • TelegramServerError (5xx) → tenacity retry with backoff

  Webhook (core.telegram.org/bots/api):
    • HTTPS only; ports 443/80/88/8443
    • secret_token: 1–256 chars, [A-Za-z0-9_-]
    • max_connections: 1..100 (default 40)
    • always specify allowed_updates explicitly

  Callback data: hard max 64 BYTES — use CallbackData factories always.
  Message text: 4096 UTF-16 code units; caption: 1024.
  Deep-link payload: 64 chars, [A-Za-z0-9_-]; base64url+HMAC for arbitrary data.
  MarkdownV2 escape chars: _*[]()~`>#+-=|{}.! — escape user input or prefer HTML.

  Mini Apps (core.telegram.org/bots/webapps):
    • initData validation: secret = HMAC_SHA256("WebAppData", bot_token);
      hash = HMAC_SHA256(sorted_key_eq_value_joined_by_\n, secret)
    • reject if auth_date older than 3600 seconds
    • never trust initDataUnsafe on client
    • JWT TTL ≤ 12h

  Telegram Stars (core.telegram.org/bots/payments-stars):
    • currency = "XTR"
    • provider_token = ""  (no payment provider needed)
    • refunds via refundStarPayment using telegram_payment_charge_id
    • digital goods ONLY

  Bot commands (core.telegram.org/bots):
    • command: 1..32 chars, [a-z0-9_]
    • description: 1..256 chars
    • use BotCommandScope* for per-audience menus
